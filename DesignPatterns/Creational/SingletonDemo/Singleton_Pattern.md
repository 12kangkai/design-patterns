# 单例模式 (Singleton Pattern)

## 1. 概念
单例模式保证一个类 **只有一个实例**，并提供一个全局访问点。  
在整个应用程序中，该类只有唯一实例存在。

---

## 2. 应用场景
- 系统中需要 **全局唯一对象**：
  - 日志记录器
  - 配置管理器
  - 数据库连接池
- 避免重复创建开销大的对象
- 控制共享资源访问

---

## 3. 单例模式的常见实现方式

| 类型 | 特点 | 线程安全 | 延迟初始化 | 适用场景 |
|------|------|----------|------------|----------|
| **饿汉式 (Eager Singleton)** | 类加载时创建实例 | 天然线程安全 | 否 | 构造开销小，应用程序启动时即可创建 |
| **懒汉式 (Lazy Singleton Unsafe)** | 调用时创建实例 | 否 | 是 | 构造开销大，单线程场景，演示线程不安全问题 |
| **双重检查锁 (Double-Check Locking)** | 线程安全，延迟初始化 | 是 | 是 | 多线程场景，需要延迟初始化 |
| **静态内部类 (Static Inner Class)** | 简洁、线程安全 | 是 | 是 | 延迟初始化，类加载安全 |
| **Lazy<T>** | C# 推荐方式，线程安全，简洁 | 是 | 是 | 所有线程安全场景，推荐使用 |

---

## 4. 注意事项
- **多线程环境**下，如果不保证线程安全，可能会出现多个实例。
- **私有构造函数**防止外部使用 `new` 创建实例。
- **序列化、反射**可能破坏单例，需要额外处理。
- **延迟初始化**可以减少开销，但需要考虑线程安全。
